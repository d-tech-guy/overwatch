# Critical Bug Fixes
Status: REQUIRED
Priority: BLOCKING

The following issues are preventing OverWatch from functioning correctly.

Do not implement workarounds.

Identify the root cause of each issue and refactor the implementation where necessary.

---

# 1. Inngest Event Dispatch Failure

## Problem

Investigations cannot start because Inngest is failing before the background workflow begins.

Current error

```
Failed to send event.

We couldn't find an event key.

Set the INNGEST_EVENT_KEY environment variable.
```

---

## Required Fix

Audit the entire Inngest configuration.

Verify

- Inngest client configuration
- Environment variables
- Event key loading
- Dev vs Production configuration
- Event sender
- Function registration
- Local development configuration

If the application is running in development,

the investigation system must still function.

If Inngest is unavailable,

the application should fail gracefully instead of crashing.

The investigation should never terminate because the event provider is unavailable.

---

## Environment Validation

Validate at startup that the following environment variables exist before creating the client.

```
INNGEST_EVENT_KEY

INNGEST_SIGNING_KEY

INNGEST_BASE_URL (if required)
```

Provide meaningful startup errors instead of runtime crashes.

---

## Resilience

The investigation pipeline should be resilient.

Expected behaviour

```
Attempt to send event

↓

Success

↓

Continue
```

If Inngest is unavailable

```
Attempt

↓

Log structured error

↓

Persist audit log

↓

Update investigation status

↓

Return meaningful error

↓

Do NOT crash the application
```

Unhandled promise rejections are unacceptable.

---

# 2. Application Approval Failure

## Problem

Institution approval is failing.

Current error

```
Error creating UUID

Found "G"

Expected UUID
```

This indicates that a UUID field is receiving a string value.

---

## Required Investigation

Audit the approval transaction.

Determine which UUID field is receiving

```
"G"

"GOD"

"GOD_EMAIL"

"GOD_CONSOLE"

or another non-UUID string.
```

Do not guess.

Trace every value passed into

```
tx.application.update()

tx.school.create()

tx.admin.create()

auditLog.create()

notification.create()
```

Verify that every UUID column receives an actual UUID.

---

## Typical Cause

Fields such as

```
reviewedBy

approvedBy

performedBy

createdBy

updatedBy

adminId

schoolId
```

must reference database UUIDs.

They must never contain

```
"GOD"

"admin"

email addresses

display names

roles

or arbitrary strings.
```

If the Platform Administrator is acting,

resolve the Platform Administrator's database record first,

then use its UUID.

---

## Approval Transaction

The approval flow should be

```
Load Application

↓

Validate Pending Status

↓

Create Auth User

↓

Create School

↓

Create Institution Administrator

↓

Resolve Platform Administrator UUID

↓

Update Application

↓

Write Audit Logs

↓

Commit
```

Every foreign key must reference an existing UUID.

---

# 3. Transaction Safety

Approval currently rolls back completely.

Keep the transaction,

but improve diagnostics.

Every failure should clearly indicate

- which operation failed
- why it failed
- which entity caused the failure

Avoid generic Prisma errors whenever possible.

---

# 4. Logging

Replace generic console output with structured logs.

Include

- Application ID
- School Name
- Transaction Stage
- Correlation ID
- Failure Reason

Never log secrets.

Never log passwords.

Never log service role keys.

---

# Success Criteria

The implementation is complete only when

✓ Investigations successfully dispatch background events.

✓ Missing Inngest configuration is detected before runtime.

✓ Missing Inngest keys never crash the application.

✓ Institution approval completes successfully.

✓ All UUID foreign keys reference valid database records.

✓ No Prisma UUID conversion errors remain.

✓ Approval transactions commit successfully.

✓ Audit logs continue to be written correctly.

✓ No unhandled promise rejections occur anywhere in the application.
